Getting started

Once you've signed up and received your public key, you're just a few steps away from integrating our WebSocket service into your project. Here's how to get started:

Step 2: Create a WebSocket Connection

Explanation:

Public Key:

The public key is used to authenticate your connection with our WebSocket service. It ensures that only authorized clients can establish a connection.

Channel ID:

The channel ID is a unique identifier for the specific communication channel you wish to connect to. It allows you to target a specific stream of data or group of users.

WebSocketClient Instance:

The `WebSocketClient` is a custom client provided by our service to simplify the process of connecting and interacting with our WebSocket server. You create an instance of this client by passing your `publicKey` and `channelId` as parameters.

Event Handlers:

`onOpen`: This event handler is triggered when the WebSocket connection is successfully established. It’s a good place to initiate any actions that should occur upon connection, such as notifying the user or sending an initial message to the server.

`onMessage`: This handler is called whenever the client receives a message from the server. You can process and respond to incoming messages within this function.

`onClose`: This event is triggered when the WebSocket connection is closed, either by the server or due to network issues. It’s important to handle this event to manage disconnections gracefully, such as attempting to reconnect or notifying the user.

`onError`: This event handles any errors that occur during the WebSocket connection lifecycle. It’s crucial for debugging and managing unexpected issues.

Connection:

The `connect()` method establishes the connection to the WebSocket server using the provided public key and channel ID. This initiates the entire WebSocket communication process.

Step 3: Send a Message to a Channel

You can send a message to a specific channel using the `sendMessage()` method provided by the WebSocketClient instance. Here’s how you can do it:

Message Format:

As you noted, `message` can be a string or any object. It’s important that the receiving end of your WebSocket handles the message format correctly, especially if you’re sending objects (like JSON). Both sender and receiver should have a consistent understanding of the message structure.

Channel ID:

The `toChannelId` should correctly identify the channel to which the message is being sent. Ensure that this ID matches the channel you intend to target.

`sendMessage()` Method

The `sendMessage()` method is used to send data to a particular channel identified by toChannelId. This method ensures that your message reaches the intended recipients.
